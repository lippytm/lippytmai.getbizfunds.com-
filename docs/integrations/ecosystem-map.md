# Ecosystem Map

## Main Role of This Repository

`lippytmai.getbizfunds.com` is the public-facing website layer for a larger connected ecosystem.

Its role is to receive traffic, present offers, capture leads, and route people into the right automation, AI, and follow-up systems.

## Core Connection Map

```text
Traffic Sources
  -> Website Pages / Landing Pages
  -> Forms / Chat / CTA Buttons
  -> MyClaw Workflows
  -> Claude Prompting / Content Support
  -> Grok Prompting / Campaign Variation
  -> Follow-up Systems / CRM / Email / Sales
```

## Website -> MyClaw

Use MyClaw for:
- lead intake
- routing by offer type
- follow-up workflows
- opportunity tagging
- internal task automation
- service request handoff

### Example Flows
- funding inquiry -> qualification -> follow-up
- automation request -> discovery intake -> service pathway
- strategy session request -> scheduling workflow -> reminders
- affiliate interest -> nurture sequence -> onboarding

## Website -> Claude

Use Claude for:
- homepage copy drafting
- landing page writing
- proposal generation
- FAQ writing
- service explanations
- email sequences
- knowledge-base structuring

## Website -> Grok

Use Grok for:
- campaign angle generation
- social post variations
- bold headline alternatives
- rapid idea branching
- category expansion
- alternative positioning experiments

## Shared Knowledge Layer

Create and maintain shared documents for:
- brand voice
- offer definitions
- customer segments
- CTA library
- funnel logic
- service descriptions
- qualification questions

These shared docs help all systems stay aligned.

## Best Practice

The website should remain the primary public hub while external tools act as support layers behind it.

That keeps branding, traffic, messaging, and conversion control inside your own ecosystem.
