# Retention Followup Routing

This document explains how active-service, nurture, and reengagement contacts should be routed after activation.

## Routing Triggers
- active service needs scheduled follow-up
- delivery context suggests expansion opportunity
- nurture hold contact becomes active again
- reengagement signal appears after inactivity

## Routing Inputs
- contact_id
- lead_stage
- service_state
- last_touch_at
- next_action
- related offer or package

## Best Practices
- keep follow-up useful and timely
- use service context to guide next best step
- route back into assistant, swarm, or commerce only when the next action is clear
- track repeat engagement as a value signal, not just contact volume
