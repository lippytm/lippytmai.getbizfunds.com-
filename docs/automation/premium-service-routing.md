# Premium Service Routing

This document explains how qualified and premium leads should move into higher-value service paths.

## Routing Triggers
- premium lead selected
- strategy lead confirms paid next step
- service package or implementation path chosen
- complex case needs higher-touch handling

## Routing Inputs
- contact_id
- lead_stage
- primary_goal
- urgency
- selected offer or package
- next_action
- service_state

## Best Practices
- keep premium routing tied to real buying or service-fit signals
- preserve context across assistant, swarm, and commerce handoffs
- keep higher-touch routes deliberate and reviewable
- use premium routing to improve clarity and service quality, not just price tiering
