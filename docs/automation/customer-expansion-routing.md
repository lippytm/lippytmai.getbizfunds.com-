# Customer Expansion Routing

This document explains how existing customers should be routed into useful next-step offers, services, or knowledge paths.

## Goal

Create more value from active relationships by matching customers to the next relevant path after delivery or activation.

## Routing Triggers
- active service is complete or stable
- delivery context suggests next-fit offer
- repeat engagement signal appears
- retention follow-up identifies a relevant expansion path

## Routing Inputs
- contact_id
- service_state
- related offer or package
- last_touch_at
- next_action
- expansion_fit

## Best Practices
- expand from real fit and service context
- keep next-step offers easy to understand
- route into assistant, swarm, knowledge, or partner paths only when useful
- use expansion outcomes to improve product and service design
