# AI Registry Audit Checklist

## Purpose
Use this checklist when reviewing the health, usefulness, and trustworthiness of shared registries.

## Ownership Review
- [ ] canonical owner is visible
- [ ] review cadence is visible
- [ ] purpose is visible

## Structure Review
- [ ] key fields are consistent
- [ ] naming is consistent
- [ ] stale entries are identifiable
- [ ] registry still matches visible source files

## Quality Review
- [ ] highest-value entries are current
- [ ] duplicated entries are minimized
- [ ] unclear entries are flagged
- [ ] missing high-value entries are visible

## Reuse Review
- [ ] registry improves discoverability
- [ ] registry helps reduce duplication
- [ ] registry helps governance or reuse decisions

## Audit Outcome
- [ ] healthy
- [ ] needs cleanup
- [ ] needs ownership clarification
- [ ] needs restructuring

## Best Practices
- keep registries lightweight but trustworthy
- audit the most reused registries first
- use audit results to improve memory, not create more clutter
