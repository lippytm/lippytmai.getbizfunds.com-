# AI System Health Improvement Playbook

## Purpose

This playbook defines how to improve overall ecosystem health across repos, CRM, assets, prompts, workflows, learning, and governance.

## Core Principle

System health improves when visibility, continuity, ownership, reuse, and next actions improve together.

## Improvement Domains

### 1. Visibility Health
Improve role clarity, purpose visibility, status freshness, and next action visibility.

### 2. Continuity Health
Improve lifecycle freshness, handoff quality, support continuity, and owner clarity.

### 3. Reuse Health
Improve asset visibility, prompt discoverability, workflow visibility, and learning promotion.

### 4. Governance Health
Improve standards alignment, ownership clarity, drift detection, and review rhythm.

### 5. Learning Health
Improve lesson capture, promotion decisions, and reuse of durable knowledge.

## Improvement Sequence
1. diagnose weakest health area
2. identify smallest highest-leverage repair
3. apply visibility or continuity fix first
4. update registry, status, or ownership surfaces if needed
5. review whether the system is now easier to scan and continue

## Best Practices
- improve system health in layers, not all at once
- fix ambiguity before adding new components
- prefer durable reusable improvements over local patchwork
- connect health improvements to customer and client experience outcomes
