# AI Repo Health Review Playbook

## Purpose

This playbook defines how AI systems and humans can review repository health in a repeatable, transparent, and actionable way.

## Core Principle

Repo health is not only about whether files exist.
It is about whether a repository is understandable, reusable, well-scoped, well-linked to the wider fleet, and easy to continue safely.

## Review Areas

### 1. Identity and Role
Review:
- role clarity
- purpose visibility
- maturity visibility
- owner visibility

### 2. Planning and Status
Review:
- status freshness
- roadmap visibility
- current priorities
- next action clarity

### 3. Interoperability
Review:
- related repos listed clearly
- inputs visible
- outputs visible
- dependencies visible

### 4. Asset Quality
Review:
- reusable assets surfaced
- prompt inventory visible
- workflows visible if relevant
- learning assets visible if relevant

### 5. Standards Alignment
Review:
- required files by role
- recommended files by role
- terminology alignment
- lifecycle alignment
- prompt naming alignment

## Suggested Review Output
Use:
- strongest areas
- weakest areas
- highest-leverage fix
- readiness for wider rollout

## Best Practices
- fix scanability first
- prefer smallest high-leverage repair first
- connect repo health to reuse and continuation quality
- use scorecards for comparison, not bureaucracy
