# AI Fleet Rollup Playbook

## Purpose

This playbook defines how AI systems and humans can produce clearer, more comparable, and more actionable fleet rollups across repos.

## Core Principle

A fleet rollup should help answer:
- what is healthy
- what is under-documented
- what is drifting
- what is blocked
- what should improve next

It should not only summarize activity. It should improve decision quality.

## Rollup Areas

### 1. Repo Health
Compare:
- role clarity
- purpose visibility
- status freshness
- next action clarity

### 2. Standards Health
Compare:
- file surface drift
- taxonomy drift
- lifecycle drift
- prompt naming drift
- ownership drift

### 3. Asset and Reuse Health
Compare:
- asset visibility
- prompt visibility
- workflow visibility
- learning asset discoverability

### 4. CRM and CX Health
Compare where relevant:
- next action quality
- satisfaction signal strength
- handoff quality
- churn risk visibility

### 5. Innovation and Learning Health
Compare:
- experiment maturity visibility
- promotion readiness
- lesson reuse
- stranded insights

## Rollup Outputs
- healthiest repos
- weakest surfaces
- strongest reusable assets
- biggest drift risks
- highest-leverage next improvements

## Best Practices
- keep rollups comparative and scannable
- include best next actions, not only observations
- make strengths visible, not only weaknesses
- use rollups to improve sequencing of upgrades across the fleet
