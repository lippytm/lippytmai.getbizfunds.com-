# AI Canonical Ownership Review Pack

## Purpose
Use this pack to help AI systems review ownership clarity for prompts, templates, standards, schemas, workflows, and learning assets.

## Review Questions
- What asset should have a single canonical home?
- What is being duplicated without clear ownership?
- What local copy should become a reference instead?
- Which repo role is the best long-term owner?
- What cleanup is needed before promotion or standardization?

## Suggested Prompt Ideas
- ai-canonical-ownership-review-v1
- ai-duplicate-asset-detection-v1
- ai-asset-home-recommendation-v1
- ai-canonical-source-visibility-review-v1
- ai-shared-asset-standardization-review-v1

## Best Practices
- prefer one canonical owner when possible
- allow local references without silent duplication
- document ownership in ASSETS files and registries
- use ownership reviews to reduce confusion, not just to reorganize for its own sake
