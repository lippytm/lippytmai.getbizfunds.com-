# AI Canonical Asset Promotion Sequence

## Purpose

This guide defines a practical order for promoting assets from local or experimental use into stronger long-term canonical homes.

## Promotion Sequence

### Step 1 — Detect Reuse Value
Identify assets that are reused, referenced, or repeatedly recreated.

### Step 2 — Confirm Role Fit
Confirm which repo role is the strongest long-term owner:
- knowledge
- front_door
- workflow
- product
- control_tower

### Step 3 — Clean Up
Improve:
- naming
- ownership visibility
- summary quality
- documentation quality
- source references

### Step 4 — Promote
Move or formally reference the asset in the stronger canonical home.

### Step 5 — Update Registries and Inventories
Refresh:
- assets registry
- prompt registry if relevant
- ASSETS surfaces
- role-specific inventories

### Step 6 — Review Reuse
Check whether the promoted asset is:
- easier to find
- easier to trust
- easier to reuse
- still current enough

## Best Practices
- promote only durable value, not every local note
- improve clarity before promotion
- keep references visible after promotion so local repos do not break continuity
- prefer one canonical home with clear references over many drifting copies
