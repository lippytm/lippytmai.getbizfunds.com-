# Fleet Governance Review Sequence

## Purpose
This guide defines a transparent review sequence for using AI and human review together to improve ecosystem health, CRM quality, documentation quality, and asset reuse.

## Review Sequence

### Step 1 — Repo Surface Review
Check role clarity, purpose visibility, status visibility, and next action visibility.

### Step 2 — Standards Alignment Review
Check missing files, naming drift, lifecycle drift, and prompt naming drift.

### Step 3 — Asset Visibility Review
Check whether prompts, templates, schemas, workflows, and guides are surfaced clearly.

### Step 4 — CRM and Relationship Review
Check lifecycle continuity, engagement quality, satisfaction risk, and next-best-action quality.

### Step 5 — Interoperability Review
Check dependencies, outputs, weak handoffs, and unclear relationships between repos.

### Step 6 — Innovation Promotion Review
Check whether lessons and validated ideas are being promoted into stronger repos.

### Step 7 — Priority Review
Compare all findings and recommend the highest-leverage next actions across the fleet.

## Best Practices
- review visibility before deep optimization
- improve canonical ownership before duplicating assets
- improve next actions before creating new complexity
- use AI for structured review and humans for priority judgment when needed
- keep outputs concise enough for practical follow-through
