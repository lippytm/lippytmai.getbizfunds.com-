# AI Handoff Improvement Playbook

## Purpose

This playbook defines how to improve handoffs between repos, workflows, humans, AI systems, CRM stages, and support or delivery steps.

## Core Principle

A handoff fails when the next layer has to guess too much.
A strong handoff preserves:
- context
- next action
- ownership
- dependencies
- risks

## Review Areas

### 1. Context Preservation
Was enough prior context carried forward?

### 2. Next Action Visibility
Is the next action explicit and practical?

### 3. Ownership Clarity
Is the next owner or responsible layer obvious?

### 4. Risk Visibility
Are trust, churn, support, or dependency risks visible enough?

### 5. Dependency Visibility
Is the next layer told what it depends on?

## Improvement Sequence
1. strengthen summary quality
2. make next action explicit
3. make ownership explicit
4. surface risks
5. surface dependencies
6. improve linked references when needed

## Best Practices
- improve summaries before adding more workflow complexity
- use handoff reviews to reduce repeated explanation
- treat weak handoffs as trust and continuity problems
- keep handoff outputs readable in under a minute
