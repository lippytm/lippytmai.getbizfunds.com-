# AI Ecosystem Maintenance Loops

## Purpose

This document defines recurring AI maintenance loops for keeping the ecosystem healthier, better documented, more reusable, and easier to upgrade over time.

---

## Loop 1 — Weekly Repo Surface Review

### Objective
Check whether important repos still have clear role, purpose, status, and next action surfaces.

### Review Questions
- Is the repo role still clear?
- Is the purpose visible quickly?
- Are status and roadmap still current enough?
- Are reusable assets surfaced clearly?

### Output
Short review summary and top cleanup actions.

---

## Loop 2 — Monthly Standards Alignment Review

### Objective
Check whether repos still align with shared taxonomy, lifecycle names, prompt naming, and role-based file expectations.

### Review Questions
- Are required files missing?
- Are naming patterns drifting?
- Are duplicated standards emerging?
- Which repos need upgrade attention first?

### Output
Standards drift summary and highest-value fixes.

---

## Loop 3 — Monthly Asset Reuse Review

### Objective
Find high-value assets that should be reused more widely.

### Review Questions
- Which prompts are most reusable?
- Which templates are underused?
- Which schemas should be copied or referenced elsewhere?
- Which repos should expose their assets more clearly?

### Output
Reuse candidates and target repos.

---

## Loop 4 — Monthly Prompt Quality Review

### Objective
Keep prompt inventories useful, versioned, and easier to scan.

### Review Questions
- Which prompts are vague?
- Which prompts are duplicated?
- Which prompt categories are missing?
- Which prompts need version upgrades?

### Output
Prompt quality improvements and naming cleanup actions.

---

## Loop 5 — Quarterly Fleet Visibility Review

### Objective
Improve control-tower quality and reduce blind spots.

### Review Questions
- Which repos still lack good status surfaces?
- Which repos have unclear dependencies?
- Which repos need role clarification?
- Which repos should be prioritized next for standards rollout?

### Output
Fleet improvement plan and new priority sequence.

---

## Loop 6 — Innovation Promotion Review

### Objective
Prevent useful experiments from being stranded.

### Review Questions
- Which experiments are mature enough to promote?
- Which should stay experimental?
- Which lessons belong in knowledge repos?
- Which ideas belong in product or front-door repos?

### Output
Promotion or retirement recommendations.

---

## Best Practices
- use recurring loops to keep the ecosystem alive, not static
- keep outputs short enough to act on
- favor practical next steps over abstract analysis
- use AI to reduce drift, duplication, and hidden confusion
