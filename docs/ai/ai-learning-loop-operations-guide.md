# AI Learning Loop Operations Guide

## Purpose

This guide defines how AI systems can capture, review, promote, and reuse learning across repos, CRM, workflows, support, and innovation.

## Learning Loop

### Step 1 — Capture
Capture lessons from:
- repo reviews
- CRM reviews
- support cases
- customer journey diagnostics
- innovation experiments
- standards drift reviews

### Step 2 — Summarize
Create clear lesson summaries with:
- what happened
- what mattered
- what should change
- where the lesson belongs

### Step 3 — Classify
Classify lessons as:
- local only
- reusable pattern
- canonical standards candidate
- product candidate
- learning asset candidate

### Step 4 — Promote
Move durable lessons into:
- knowledge repos
- front-door assets
- workflow inventories
- product guidance
- standards docs

### Step 5 — Review Reuse
Check whether promoted lessons are:
- easier to find
- actually reused
- still current enough

## Best Practices
- capture lessons in small structured formats
- promote durable learning into stronger homes
- avoid letting learning stay hidden in one-off notes
- use registries and inventories to preserve reuse visibility
