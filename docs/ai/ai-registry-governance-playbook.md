# AI Registry Governance Playbook

## Purpose

This playbook defines how shared registries should be governed so they remain useful, current, and trustworthy across the ecosystem.

## Core Principle

Registries are part of structured memory.
If they drift, become stale, or lose ownership clarity, the wider AI ecosystem becomes harder to trust and harder to improve.

## Registry Types Covered
- prompt registries
- assets registries
- fleet indexes
- KPI registries
- self-healing registries
- learning promotion registries
- CRM registries

## Governance Questions
- Who owns this registry?
- How often should it be reviewed?
- Which fields must remain consistent?
- What makes an entry stale?
- When should an entry be promoted, updated, or retired?

## Governance Sequence
1. confirm canonical owner
2. confirm required fields and naming consistency
3. review stale or unclear entries
4. compare registry to visible source files
5. update or retire weak entries
6. surface best next cleanup actions

## Best Practices
- keep registries lightweight but consistent
- review high-value registries on a regular cadence
- use registries to improve memory and governance, not add unnecessary overhead
- keep ownership and review cadence visible
