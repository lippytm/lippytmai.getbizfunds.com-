# AI Asset Reuse Operations Guide

## Purpose

This guide defines how AI systems and humans can improve asset reuse across prompts, templates, schemas, workflows, guides, and learning materials.

## Core Principle

Asset reuse improves when the ecosystem makes high-value assets visible, well-scoped, well-owned, and easy to continue.

## Reuse Workflow

### Step 1 — Detect
Find assets that are:
- reused often
- duplicated often
- hard to find
- strong but underexposed

### Step 2 — Classify
Classify assets as:
- local only
- reusable within role
- reusable fleet-wide
- canonical standards candidate

### Step 3 — Improve Visibility
Improve:
- ASSETS surfaces
- prompt inventories
- workflow inventories
- learning inventories
- registry entries

### Step 4 — Clarify Ownership
Make clear:
- canonical owner
- allowed references
- local variations vs canonical source

### Step 5 — Promote or Standardize
Promote durable assets into:
- knowledge repos
- front-door assets
- workflow libraries
- product libraries
- control-tower references when relevant

## Best Practices
- improve visibility before forcing standardization
- promote only assets with durable reuse value
- use registries and inventories to reduce hidden asset drift
- keep canonical ownership visible to reduce confusion
