# Repo Audit Checklist

## Purpose
Use this checklist to review whether a repository is aligned with the fleet standards and interoperable enough to work smoothly with the rest of the ecosystem.

## Identity and Role
- [ ] Repo purpose is clear
- [ ] Repo role is declared
- [ ] Repo owner is identified
- [ ] Repo maturity is identified
- [ ] Last reviewed date is present

## Standard Files
- [ ] README.md exists and is useful
- [ ] CAPABILITIES.md exists
- [ ] INTEGRATIONS.md exists
- [ ] STATUS.md exists
- [ ] ROADMAP.md exists
- [ ] PROMPTS.md exists if prompts are relevant

## Interoperability
- [ ] Related repos are named
- [ ] Inputs from other repos are documented
- [ ] Outputs to other repos are documented
- [ ] Shared standards are identified
- [ ] Reusable assets are exposed clearly

## Standards Alignment
- [ ] Shared taxonomy is followed
- [ ] Shared lifecycle names are followed
- [ ] Shared prompt naming is followed
- [ ] File names are consistent enough for scanning

## Operational Quality
- [ ] Current priorities are clear
- [ ] Risks or blockers are visible
- [ ] Suggested next action is visible
- [ ] Status is recent enough to trust

## Reuse Potential
- [ ] Repo provides reusable docs, prompts, templates, schemas, or workflows
- [ ] Repo does not duplicate shared assets without reason
- [ ] Repo points to canonical sources when another repo owns the standard

## Audit Outcome
Choose one:
- [ ] strong and ready for fleet adoption
- [ ] usable but needs cleanup
- [ ] partial and needs standards work
- [ ] not ready yet

## Recommended Next Steps
- standard files to add
- outdated files to refresh
- shared terms to normalize
- interoperability links to improve
- reusable assets to expose
