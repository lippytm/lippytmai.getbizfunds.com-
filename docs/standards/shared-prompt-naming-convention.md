# Shared Prompt Naming Convention

## Purpose
This guide standardizes prompt names so they can be reused, searched, versioned, and linked across repositories.

## Naming Pattern
Use:
`domain-purpose-mode-version`

Examples:
- crm-intake-type-v1
- support-triage-hybrid-v1
- website-router-type-v2
- innovation-brainstorm-guided-v1
- education-roadmap-type-v1

## Versioning Guidance
- use v1 for first stable version
- increment version when prompt behavior or output expectations materially change
- document canonical source repo when prompts are shared

## Best Practices
- keep names readable and machine-friendly
- do not rely on vague names like prompt1 or final_prompt
- reuse domain and purpose labels consistently across repos
