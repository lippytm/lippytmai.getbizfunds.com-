# Repo Upgrade Checklist

## Purpose
Use this checklist when improving an older repository so it matches the shared fleet standards more closely.

## Discovery
- [ ] review current README
- [ ] identify repo role
- [ ] identify owner
- [ ] identify related repos
- [ ] identify missing standard files
- [ ] identify outdated files

## Standardization
- [ ] add or update CAPABILITIES.md
- [ ] add or update INTEGRATIONS.md
- [ ] add or update STATUS.md
- [ ] add or update ROADMAP.md
- [ ] add or update STANDARDS.md
- [ ] add or update PROMPTS.md if relevant
- [ ] add or update ASSETS.md if relevant

## Alignment
- [ ] align taxonomy terms
- [ ] align lifecycle names
- [ ] align prompt naming where relevant
- [ ] define current priorities
- [ ] define next best action

## Interoperability
- [ ] document inputs from other repos
- [ ] document outputs to other repos
- [ ] identify reusable assets
- [ ] identify duplicated assets that should point to a single source

## Final Review
- [ ] repo purpose is clear
- [ ] status is clear
- [ ] roadmap is clear
- [ ] integration surface is clearer than before
- [ ] repo is more useful to the wider ecosystem
