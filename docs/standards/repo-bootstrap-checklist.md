# Repo Bootstrap Checklist

## Purpose
Use this checklist when starting a new repository so it begins with the shared operating skeleton.

## Bootstrap Steps
- [ ] create README.md
- [ ] add CAPABILITIES.md
- [ ] add INTEGRATIONS.md
- [ ] add STATUS.md
- [ ] add ROADMAP.md
- [ ] add STANDARDS.md
- [ ] add PROMPTS.md if prompts are relevant
- [ ] add ASSETS.md if reusable assets are relevant
- [ ] add CHANGELOG.md
- [ ] add interoperability manifest yaml

## Role Setup
- [ ] declare repo role
- [ ] declare maturity level
- [ ] identify owner
- [ ] identify related repos
- [ ] identify primary users and outcomes

## Standards Setup
- [ ] align shared taxonomy
- [ ] align lifecycle naming
- [ ] align prompt naming if needed
- [ ] define current priorities
- [ ] define suggested next action

## Interoperability Setup
- [ ] document inputs from other repos
- [ ] document outputs to other repos
- [ ] identify reusable assets
- [ ] identify main dependency

## Review Check
- [ ] purpose is clear near the top
- [ ] repo can be scanned quickly by humans and AI
- [ ] standard files are in place
