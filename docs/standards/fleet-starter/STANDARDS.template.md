# STANDARDS

## Purpose
Describe the shared standards this repository follows and any local exceptions.

## Shared Standards Followed
- shared taxonomy reference
- shared lifecycle naming guide
- shared prompt naming convention
- interoperability manifest template
- fleet starter file pattern

## Required Standard Files
- README.md
- CAPABILITIES.md
- INTEGRATIONS.md
- STATUS.md
- ROADMAP.md

## Optional Standard Files
- PROMPTS.md
- ASSETS.md

- CRM.md

## Naming Rules
- use snake_case for machine-friendly stage and key names where practical
- use readable markdown file names for human navigation
- use version markers for prompts and schemas when behavior changes materially

## Review Cadence
- active repos: monthly review minimum
- stable repos: quarterly review minimum
- fast-changing repos: review as needed plus monthly snapshot

## Local Exceptions
Document any deviations from shared standards and why they exist.

## Best Practices
- standardize the visible surfaces first
- preserve flexibility in internal implementation
- keep exceptions explicit rather than hidden
