# CHANGELOG

## Purpose
Track meaningful changes to this repository's standards, assets, workflows, or major capabilities.

## Format Guidance
Use simple dated entries.

## Entries
### YYYY-MM-DD
- added:
- updated:
- removed:
- notes:

## Best Practices
- log changes that affect other repos or shared assets
- keep entries concise and useful
- use changelog notes to support cross-repo awareness
