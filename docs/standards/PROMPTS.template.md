# PROMPTS

## Purpose
Document the prompt assets this repository owns, reuses, or depends on.

## Prompt Naming Convention
Use a structure such as:
`domain-purpose-mode-version`

Examples:
- crm-intake-type-v1
- support-triage-hybrid-v1
- website-router-type-v1
- innovation-brainstorm-guided-v1

## Prompt Quality Rules
- every prompt should have a clear purpose
- every prompt should state expected outputs
- every prompt should be versioned when materially changed
- shared prompts should identify canonical source repo when applicable

## Best Practices
- do not let prompt names become random
- keep prompt scope narrow enough to test and improve
- version prompts when output behavior changes meaningfully
