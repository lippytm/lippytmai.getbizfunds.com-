# Role to File Mapping

## Purpose
This guide shows which files are required, recommended, or optional for each repo role.

## Usage
Use this file when deciding what to stamp into a real repo during bootstrap or upgrade.

## File Legend
- **Required** = should exist for this role
- **Recommended** = strongly useful for this role
- **Optional** = add when relevant

- add recommended files in the next pass
- use optional files only where they create real value
- do not overload small repos with unnecessary files
