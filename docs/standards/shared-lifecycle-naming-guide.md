# Shared Lifecycle Naming Guide

## Purpose
This guide helps repositories use the same lifecycle language so humans and AI can move across repos with less confusion.

## High-Level Lifecycle
- new
- engaged
- qualified
- booked
- active
- nurture
- at_risk
- advocate
- inactive
- reactivated

## Pipeline Family Guidance
### Lead Intake
Use names such as:
- new
- welcomed
- intent_identified
- qualified
- ready_for_booking
- nurture
- dormant

### Sales
Use names such as:
- discovery
- offer_matched
- call_booked
- proposal_or_plan
- decision_pending
- won
- lost
- deferred

### Onboarding
Use names such as:
- welcome_sent
- intake_requested
- intake_complete
- assets_received
- setup_in_progress
- first_milestone
- fully_active

### Support
Use names such as:
- new_issue
- triage
- in_progress
- waiting_on_user
- resolved
- follow_up_complete

### Retention
Use names such as:
- new_client
- stable
- growth_opportunity
- at_risk
- renewed
- paused
- closed

## Best Practices
- use snake_case for machine-friendly names
- keep stage names short and decision-oriented
- do not invent new lifecycle names when a shared one already exists
- document exceptions locally if a repo needs role-specific stages
