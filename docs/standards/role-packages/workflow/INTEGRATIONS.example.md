# INTEGRATIONS

## Purpose
This repository connects routing logic, automation patterns, and operational execution with the wider ecosystem.

## Integration Summary
This repository mainly acts as a:
- workflow engine
- reusable execution layer
- integration support source
- operations helper

## Inputs From Other Repos
| Source Repo | What Comes In | Why It Matters |
|---|---|---|
| front-door repos | requests, routed needs, public signals | turns public demand into internal flows |
| knowledge repos | SOPs, prompts, standards | improves consistency and reuse |
| product repos | delivery requirements | supports product execution |

## Outputs To Other Repos
| Target Repo | What Goes Out | Why It Matters |
|---|---|---|
| front-door repos | routing rules, automation paths | improves handoff quality |
| product repos | workflow assets, execution support | speeds delivery patterns |
| control_tower repos | status signals, reusable patterns | improves fleet visibility |

## Shared Standards Followed
- shared taxonomy
- shared lifecycle names
- shared prompt naming convention when prompts are relevant
- interoperability manifest template

## Integration Rules
- expose reusable workflow logic clearly
- separate public routing from internal execution details
- keep links to canonical SOP or standards sources
- keep repeated execution patterns portable

2. improve handoff clarity between front-door and product repos
3. improve workflow status visibility for fleet coordination
