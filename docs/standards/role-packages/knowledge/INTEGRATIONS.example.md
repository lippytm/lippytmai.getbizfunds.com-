# INTEGRATIONS

## Purpose
This repository connects shared prompts, standards, SOPs, templates, and learning assets with the wider ecosystem.

## Integration Summary
This repository mainly acts as a:
- knowledge source
- standards source
- reusable asset source
- learning support layer

## Inputs From Other Repos
| Source Repo | What Comes In | Why It Matters |
|---|---|---|
| front-door repos | public-facing needs and guidance gaps | helps shape practical knowledge assets |
| workflow repos | execution patterns and SOP needs | improves useful standards and templates |
| product repos | implementation lessons and prompts | improves reusable guidance |

## Outputs To Other Repos
| Target Repo | What Goes Out | Why It Matters |
|---|---|---|
| front-door repos | public-friendly guidance and prompts | improves clarity and routing |
| workflow repos | SOPs and standards | improves consistency |
| product repos | prompts, templates, playbooks | improves repeatability |
| control_tower repos | standards status and audit guidance | improves governance visibility |

## Shared Standards Followed
- shared taxonomy
- shared lifecycle names
- shared prompt naming convention
- interoperability manifest template

## Integration Rules
- keep canonical ownership of shared knowledge assets visible
- reduce duplicated standards across repos
- expose the most reusable assets clearly
- connect lessons learned back into improved guidance

## Next Integration Priorities
1. improve discoverability of reusable assets
2. reduce duplication across repos
3. improve standards visibility for fleet coordination
