# INTEGRATIONS

## Purpose
This repository connects status, standards, rollups, and governance signals across the wider repository fleet.

## Integration Summary
This repository mainly acts as a:
- coordinator
- reporting source
- standards observer
- rollup surface

## Inputs From Other Repos
| Source Repo | What Comes In | Why It Matters |
|---|---|---|
| front-door repos | public-facing status and priorities | visibility into entry-point health |
| workflow repos | operations state | visibility into execution layers |
| product repos | product or delivery state | visibility into implementation layers |
| knowledge repos | standards changes and reusable assets | standards alignment |

## Outputs To Other Repos
| Target Repo | What Goes Out | Why It Matters |
|---|---|---|
| front-door repos | fleet signals and references | stronger ecosystem navigation |
| workflow repos | standards reminders or review findings | better alignment |
| product repos | status expectations and rollup visibility | stronger reporting |
| knowledge repos | fleet findings and standards needs | better shared guidance |

## Shared Standards Followed
- shared taxonomy
- shared lifecycle names
- shared status template
- fleet rollup template
- interoperability manifest template

## Integration Rules
- do not replace repo-level ownership of details
- summarize clearly enough for fleet scanning
- prefer rollups over duplicated deep docs
- flag missing status or missing standards surfaces
