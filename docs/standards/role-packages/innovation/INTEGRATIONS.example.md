# INTEGRATIONS

## Purpose
This repository connects experiments, prototypes, and early lessons with the wider ecosystem.

## Integration Summary
This repository mainly acts as a:
- experiment layer
- prototype source
- lesson generator
- future capability feeder

## Inputs From Other Repos
| Source Repo | What Comes In | Why It Matters |
|---|---|---|
| front-door repos | unmet needs and opportunity signals | helps choose useful experiments |
| workflow repos | execution constraints | keeps experiments grounded |
| product repos | real implementation needs | helps test practical ideas |
| knowledge repos | standards and prompt references | keeps tests understandable |

## Outputs To Other Repos
| Target Repo | What Goes Out | Why It Matters |
|---|---|---|
| knowledge repos | lessons learned | improves reusable guidance |
| product repos | validated concepts | helps future implementation |
| front-door repos | proven engagement ideas | may improve public experience |
| control_tower repos | maturity and status signals | improves visibility |

## Shared Standards Followed
- shared taxonomy
- shared prompt naming convention
- interoperability manifest template

## Integration Rules
- keep experiments understandable enough to hand off later
- move durable lessons into stronger long-term repos
- avoid letting temporary tests become hidden permanent dependencies
- mark maturity clearly

## Next Integration Priorities
1. improve capture of lessons learned
2. improve movement of validated ideas into stronger repos
3. improve visibility of experiment maturity
