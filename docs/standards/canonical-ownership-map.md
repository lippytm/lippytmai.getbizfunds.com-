# Canonical Ownership Map

## Purpose
Use this map to clarify which repo role should normally own which shared asset types.

## Ownership Guidance

### Knowledge Role Normally Owns
- shared standards docs
- prompt naming conventions
- shared templates
- learning guides
- playbooks
- canonical reusable prompts

### Front-Door Role Normally Owns
- public-facing routing guidance
- CRM entry assets
- top-level relationship prompts
- public navigation surfaces

### Workflow Role Normally Owns
- reusable workflow patterns
- routing and execution logic
- workflow-specific checklists
- handoff patterns

### Product Role Normally Owns
- product or service implementation assets
- delivery guides
- setup checklists
- product-specific prompts

### Control-Tower Role Normally Owns
- fleet rollups
- audit summaries
- coordination views
- fleet visibility surfaces

### Innovation Role Normally Owns
- experiments
- prototype prompts
- lesson capture notes
- concept validation materials

## Best Practices
- use one canonical home when possible
- allow local references without duplicating ownership silently
- promote mature assets out of innovation repos into stronger long-term homes
- keep ownership visible in ASSETS files and registries
